The **Rebase Client SDK** (`@rebasepro/client`) is a lightweight, transport-agnostic SDK that exposes a beautiful, fluent API modeled closely after Firebase and Supabase. You can plug it into any JavaScript environment—React, Vue, Node.js, React Native, Svelte, or Vanilla JS.

It deeply unifies your access to Rebase functionalities:
* **Authentication:** Automatically maintains your token session (incl. background refreshes).
* **Data Sources (HTTP):** Exposes a typed, chainable query builder to interact with your data.
* **Realtime (WebSocket):** Automatically reacts to authentication state changes and mirrors realtime changes.
* **Storage:** Unified bucket/path abstraction to easily upload and resolve files natively.
* **Serverless RPC:** Quickly invoke generic backend endpoints with automatic authorization via `client.call()`.

---

## 🚀 Installation

```bash
npm install @rebasepro/client @rebasepro/types
```

## 🛠 Initialization

The client acts as your primary gateway. You initialize it once and reuse it across your application. 

```typescript
import { createRebaseClient } from "@rebasepro/client";

// Optionally import your generated database types for end-to-end type safety!
import type { Database } from "./generated-types"; 

export const rebase = createRebaseClient<Database>({
    baseUrl: process.env.VITE_REBASE_ENDPOINT,
    websocketUrl: process.env.VITE_REBASE_WS_ENDPOINT,

    // Optional override for Node.js SSR environments:
    // WebSocket: require('ws') 
});
```

Because we've typed `createRebaseClient<Database>`, every query you write against it will now be strictly validated by TypeScript!

## 🔒 Authentication

The auth subsystem cleanly mirrors what you know from other backend-as-a-service providers. Its state automatically propagates to the underlying Data and Realtime layers.

```typescript
// 1. Sign In
const { user, accessToken } = await rebase.auth.signInWithEmail("user@example.com", "password");

// 2. Fetch Session synchronously
const session = rebase.auth.getSession();

// 3. Listen to auth state changes cleanly
rebase.auth.onAuthStateChange(async (event, session) => {
    if (event === "SIGNED_IN") {
        console.log("Welcome!", session.user);
    } else if (event === "SIGNED_OUT") {
        console.log("Goodbye!");
    }
});

// 4. Sign Out
await rebase.auth.signOut();
```
> [!NOTE]
> When `onAuthStateChange` detects a logout or login, the WebSocket natively drops the connection and rebinds context, ensuring secure access to your real-time topics instantly.

## 🔍 Fluent Data Querying

Querying data is intuitive and chainable. Access any generated collection directly off the client and stack filters beautifully. 

### Fetching Data

```typescript
const posts = await rebase.collection("posts")
    .where("status", "==", "published")
    .where("views", ">", 100)
    .orderBy("createdAt", "desc")
    .limit(20)
    .find();
```

### Mutating Data

Inserting, updating, and deleting are straightforward and typed. 

```typescript
// Insert
const newPost = await rebase.collection("posts").save({
    title: "Understanding Rebase SDK",
    status: "draft"
});

// Edit existing
await rebase.collection("posts").save({
    id: "some_uuid",
    title: "Understanding Rebase SDK (Updated)"
});

// Delete
await rebase.collection("posts").delete("some_uuid");
```

## ⚡ Real-Time Subscriptions

For real-time applications, use the `.listen()` method instead of `.find()`. The WebSocket handles the connection pooling, reconnection policies, and auth-bindings automatically behind the scenes.

```typescript
const unsubscribe = rebase.collection("posts")
    .where("status", "==", "published")
    .listen(
        // On Updates (Initial payload + partial differentials)
        (response) => {
            console.log("Live posts update:", response.data);
        },
        // On Pipeline Errored
        (error) => {
            console.error("Realtime subscription crashed", error);
        }
    );

// Stop listening when the component unmounts
unsubscribe();
```

## 📦 Storage

Interacting with your file storage acts as a natural extension of your client instance. The backend's active Storage Driver interprets your paths invisibly.

```typescript
const file = new File(["dummy content"], "hello.txt", { type: "text/plain" });

// Upload File
await rebase.storage.uploadFile({
    file,
    path: "users",
    fileName: "my_profile_pic.png",
    onProgress: (progress) => {
        console.log(`Uploading... ${progress}%`);
    }
});

// Resolve Public/Signed URL
const url = await rebase.storage.getDownloadURL("users/my_profile_pic.png");
```

## 🔌 Custom Endpoint RPC

If you have custom backend endpoints configured (e.g., executing a serverless function, trigger, or bespoke query), you can call them directly with automatic `Authorization` header injection using `client.call()`.

```typescript
// Triggers an authorized POST to {baseUrl}/my-custom-endpoint 
const result = await rebase.call<{ success: boolean }>("/my-custom-endpoint", { 
    triggerCustomJob: true 
});

console.log(result.success);
```
