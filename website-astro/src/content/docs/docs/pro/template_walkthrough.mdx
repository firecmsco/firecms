When you run `npx create-rebase-app`, a project with three packages is created:

```
your-project/
├── frontend/     # React SPA (Vite + TypeScript)
├── backend/      # Node.js + Express API server
└── shared/       # Collections schema, shared between front and backend
```

This page walks through the key parts of each.

---

## Shared: Collections

The `shared/collections/` folder is where your data schema lives. Collections are TypeScript objects that define both the UI shape and the database shape of your data.

Each collection maps to a Postgres table and drives:
- The **admin UI** (table view, form fields, filters)
- The **database schema** (columns, types, constraints)
- The **RLS policies** (row-level security rules)
- The **REST and GraphQL APIs**

A collection file looks like this:

```ts
// shared/collections/posts.ts
import { EntityCollection } from "@rebasepro/types";

const postsCollection: EntityCollection = {
    name: "Posts",
    singularName: "Post",
    slug: "posts",     // URL path in the CMS: /c/posts
    dbPath: "posts",   // Postgres table name
    icon: "Article",
    properties: {
        title: {
            name: "Title",
            type: "string",
            validation: { required: true }
        },
        status: {
            name: "Status",
            type: "string",
            enum: [
                { id: "draft", label: "Draft", color: "grayLight" },
                { id: "published", label: "Published", color: "greenLight" }
            ]
        },
        author: {
            name: "Author",
            type: "relation",
            relationName: "author"
        }
    },
    relations: [
        {
            relationName: "author",
            target: () => authorsCollection,
            cardinality: "one",
            direction: "owning"
        }
    ],
    securityRules: [
        {
            name: "Enable read access for all users",
            operation: "select",
            mode: "permissive",
            using: "true",
            roles: ["public"]
        }
    ]
};

export default postsCollection;
```

Collections are exported from `shared/collections/index.ts` and consumed by both the frontend and backend.

## Backend: `backend/src/index.ts`

The backend is an **Express + Node.js** server. It handles authentication, data access, file storage, and exposes REST and GraphQL APIs.

### Database connection

```ts
import { createPostgresDatabaseConnection, initializeRebaseBackend } from "@rebasepro/backend";
import { tables, enums, relations } from "./schema.generated";

const db = createPostgresDatabaseConnection(process.env.DATABASE_URL);
```

`schema.generated.ts` is auto-generated from your collections by Drizzle ORM. You regenerate it by running `pnpm migrate`.

### Starting the server

```ts
const backend = await initializeRebaseBackend({
    collectionsDir: path.resolve(__dirname, "../../shared/collections"),
    server,
    app,
    datasource: {
        "(default)": {
            connection: db,
            schema: { tables, enums, relations },
            adminConnectionString: process.env.ADMIN_CONNECTION_STRING
        }
    },
    auth: {
        jwtSecret: process.env.JWT_SECRET,
        google: process.env.GOOGLE_CLIENT_ID
            ? { clientId: process.env.GOOGLE_CLIENT_ID }
            : undefined,
        seedDefaultRoles: true,
        allowRegistration: process.env.ALLOW_REGISTRATION === "true"
    },
    storage: {
        type: "local",
        basePath: path.resolve(__dirname, "../../uploads")
    }
});
```

`initializeRebaseBackend` automatically mounts:
- **WebSocket** server for real-time sync
- **Auth routes** (`/api/auth/*`) — login, refresh, Google OAuth
- **Storage routes** for file uploads
- **Admin routes** for schema management

### REST and GraphQL API (optional)

```ts
await initializeRebaseAPI(app, backend, {
    basePath: "/api",
    collectionsDir: path.resolve(__dirname, "../../shared/collections"),
    enableGraphQL: true,
    enableREST: true
});
```

Once enabled, your tables are instantly available at:
- `GET /api/posts` — list posts
- `POST /api/posts` — create a post
- `GET /api/graphql` — GraphQL endpoint
- `GET /api/swagger` — Swagger UI docs

## Frontend: `frontend/src/App.tsx`

The frontend is a **React + Vite** SPA. The generated `App.tsx` wires together all the controllers and renders the CMS.

### Controllers

```tsx
// Auth controller — handles login, JWT, Google OAuth
const authController = useRebaseAuthController({
    apiUrl: API_URL,
    googleClientId: GOOGLE_CLIENT_ID
});

// Data source — connects to Postgres via WebSocket for real-time sync
const postgresDelegate = usePostgresClientDataSource({
    websocketUrl: API_URL.replace(/^http/, "ws"),
    getAuthToken: authController.getAuthToken
});

// Storage source — uploads/downloads through the backend API
const storageSource = useBackendStorageSource({
    apiUrl: API_URL,
    getAuthToken: authController.getAuthToken
});

// User management — roles and permissions stored in the database
const userManagement = useBackendUserManagement({
    apiUrl: API_URL,
    getAuthToken: authController.getAuthToken,
    currentUser: authController.user
});
```

### Collections

Collections are loaded from the shared package via a Vite virtual module:

```tsx
import { collections } from "virtual:rebase-collections";
```

This import is resolved at build time by the Rebase Vite plugin, which reads `shared/collections/index.ts`.

### Studio views (dev mode)

The app exposes three developer views, only accessible in **Studio mode** (toggled with a keyboard shortcut):

```tsx
const devViews: CMSView[] = [
    {
        slug: "sql",
        name: "SQL Console",
        group: "Database",
        view: <SQLEditor />
    },
    {
        slug: "rls",
        name: "RLS Policies",
        group: "Database",
        view: <RLSEditor />
    },
    {
        slug: ["schema", "schema/*"],
        name: "Edit collections",
        group: "Schema",
        view: <CollectionsStudioView configController={collectionConfigController} />
    }
];
```

### Rendering

```tsx
return (
    <SnackbarProvider>
        <ModeControllerProvider value={modeController}>
            <Rebase
                navigationStateController={navigationStateController}
                authController={authController}
                dataSource={postgresDelegate}
                storageSource={storageSource}
                ...
            >
                {({ loading }) => {
                    if (loading || authController.initialLoading)
                        return <CircularProgressCenter />;

                    if (!authController.user)
                        return <RebaseLoginView authController={authController} ... />;

                    return (
                        <RebaseRoutes>
                            <Route element={
                                <Scaffold>
                                    <AppBar />
                                    <Drawer title={"Rebase"} />
                                    <Outlet />
                                    <SideDialogs />
                                </Scaffold>
                            }>
                                <Route path={"/"} element={<ContentHomePage />} />
                                <Route path={"/c/*"} element={<RebaseRoute />} />
                                <Route path={"/users"} element={<UsersView userManagement={userManagement} />} />
                                <Route path={"/roles"} element={<RolesView userManagement={userManagement} />} />
                                {/* dev views */}
                                <Route path={"*"} element={<NotFoundPage />} />
                            </Route>
                        </RebaseRoutes>
                    );
                }}
            </Rebase>
        </ModeControllerProvider>
    </SnackbarProvider>
);
```

### Key components

| Component | Purpose |
|-----------|---------|
| `RebaseRoutes` | Router wrapper (uses `react-router-dom` internally) |
| `RebaseRoute` | Renders a collection view at `/c/:collection/*` |
| `Scaffold` | Layout shell with sidebar + content area |
| `SideDialogs` | Slide-over panels for entity editing |
| `RebaseLoginView` | Login page with email/password and optional Google login |
| `ContentHomePage` | Default home page in CMS mode |
| `StudioHomePage` | Home page in Studio mode |
