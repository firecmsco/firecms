## Using the **Rebase PRO starter template**

The easiest way to use Rebase with Next.js is to use the **Rebase PRO starter template**. This template includes a Next.js project with Rebase already configured.

You can create a new project using the **Rebase PRO template** by running:

```bash
npx create-rebase-app
```
or

```bash
yarn create rebase-app
```

and select the **`Rebase PRO with Next.js frontend`** template.

Then follow the instructions on the screen to create your project.

### What you get

The code that will be generated for you is a Next.JS project split in **3 parts**:
- A **Rebase instance** to manage your data.
- A **frontend app** that implements CRUD functionality for a products collection, as well as a blog view.
- A **common folder** with shared components.

## Setting up **Rebase** with **Next.js** manually

You can use Rebase with Next.js. Rebase is a **React library**, so you can use it with any React framework.

In the case of Next.js, you are restricted to running Rebase on the **client side**, as Next.js does not support server side rendering of some of the React components used by Rebase.

Let's build an app using Rebase and Next.js, with the app router configured to delegate all the routes starting with `/cms` to **Rebase**.

### Create a **Next.js project**

Start by creating your Next.js project:

```bash
npx create-next-app@latest
```

Select:
- **TypeScript** as the language
- **ESLint** as the linter
- **Tailwind CSS** as the CSS framework
- **src** as the root directory
- Yes to the **app router** prompt
- Yes to customize the **default import alias** (optional)

### Install **Rebase**

Then we are going to install **Rebase PRO**. Note that we will not be adding all the plugins like the collection editor or data enhancement, but you can add them as needed.

Then install Rebase and its dependencies:

```bash
yarn add firebase@^10 @rebasepro/core@^3.0.0-beta @rebasepro/firebase@^3.0.0-beta @rebasepro/editor@^3.0.0-beta react-router@^6 react-router-dom@^6 @tailwindcss/typography typeface-rubik @fontsource/jetbrains-mono
```

Now let's import the tailwind config of Rebase. Add the **Rebase preset** `tailwind.config.js`, as well as the content paths to Rebase source code, so the right tailwind classes are picked.

```js
import rebaseConfig from "@rebasepro/ui/tailwind.config.js";
import type { Config } from "tailwindcss";

const config: Config = {
  presets: [rebaseConfig],
  content: [
    "./src/pages/**/*.{js,ts,jsx,tsx,mdx}",
    "./src/components/**/*.{js,ts,jsx,tsx,mdx}",
    "./src/app/**/*.{js,ts,jsx,tsx,mdx}",
    "./src/cms/**/*.{js,ts,jsx,tsx,mdx}",
    "./node_modules/@rebasepro/**/*.{js,ts,jsx,tsx}"
  ]
};
export default config;

```

### Disable **yarn pnp** (optional)

We prefer disabling yarn pnp for this project. You can do this by creating the file `.yarnrc` in the root of your project with the following content:

```bash
nodeLinker: node-modules
```

### Configuring the **App router**

Next.js uses a file-based router. In this guide, we will be creating the Rebase app in the `/cms` route, but you can customize this to your needs.

Rebase uses **react-router**, so we need to configure Next.js to delegate all the routes starting with `/cms` to **Rebase**.

In our `app` folder, we create a folder called `cms` and inside it another one called `[[...path]]`. This will match any route starting with `/cms`.

Then create the file `cms/[[...path]]/page.tsx` with the following content:
:::important
If you are not running Rebase in the root path of your app, you need to set the `basePath` prop to the path where you are running it. In this case, we are running it in `/cms`.
:::

```tsx
"use client";
import { RebaseApp } from "@/cms/RebaseApp";
import { RebaseRouter } from "@rebasepro/core";

export default function CMS() {
    return <RebaseRouter basePath={"/cms"}>
        <RebaseApp/>
    </RebaseRouter>;
}

```

### Creating the **CMS**

Now let's create the Rebase components. Create the file `./src/cms/RebaseApp.tsx` with the following content. Remember to replace the `firebaseConfig` with your own Firebase configuration.

import { Code } from '@astrojs/starlight/components';
import importedCode from '../../samples/samples_v3/nextjs.tsx.txt?raw';

<Code code={importedCode} lang="tsx" />

### Import the default **Rebase styles**

Create a file called `index.css` in the `./src/cms` folder with the following content:

```css
@import "@rebasepro/ui/index.css" layer(base);
@tailwind base;
@tailwind components;
@tailwind utilities;

:root {
    --color-primary: #0070F4;
    --color-secondary: #FF5B79;
}

a {
    @apply text-blue-600 dark:text-blue-400 dark:hover:text-blue-600 hover:text-blue-800
}
```

### Run

Then simply run:

```
yarn dev
```
and navigate to `http://localhost:3000/cms` to see your **Rebase** app running.

### Some considerations

- Images are loaded differently in **Next.js**.
You get a `StaticImageData` instead of the image URL (as in **vite**).
You can use it in Rebase components that expect a URL like, using the `src` property:
```tsx
import logo from "./logo.png";

<FirebaseLoginView
    logo={logo.src}/>
```
