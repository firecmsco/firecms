:::important
Rebase is an **open-source Postgres admin panel and headless CMS** built with React and TypeScript. It connects directly to your **PostgreSQL** database to provide a powerful, intuitive, and type-safe interface for managing your data.
:::

Rebase is designed to be the fastest way to build a production-ready back-office on top of Postgres. It generates full CRUD (Create, Read, Update, Delete) views from your TypeScript collection definitions, supports **relations**, **Row Level Security (RLS)**, and can be set up in minutes.

## What Can You Build?

Rebase is flexible enough to be the admin panel for any Postgres-backed project. Common use cases include:

  * **Custom Admin Panels:** Create a type-safe UI for managing your application's data (e.g., users, products, orders) without the frontend hassle.
  * **Internal Tools:** Build bespoke tools for your operations, sales, or support teams to manage workflows, track data, and run internal processes.
  * **Headless CMS:** Power your websites, mobile apps, and marketing content from a single, powerful admin interface backed by Postgres.

## Core Developer Features

Rebase is built by developers, for developers.

### 1\. Schema as Code

Define your data models as TypeScript collections. Each collection maps to a Postgres table and supports typed properties, relations, enums, and security rules. Your code becomes the single source of truth, enabling version control, type-safety, and a seamless git-based workflow.

### 2\. Instant CRUD UI

Rebase reads your schema and automatically generates a full-featured, performant, and beautiful CRUD interface with tables, forms, validation, relations, and more.

### 3\. Real-time Sync

Rebase uses WebSockets to provide real-time data synchronization. When data changes in Postgres, your admin panel updates instantly.

### 4\. Radical Extensibility

If you can build it in React, you can build it in Rebase. Create custom form fields, entirely new top-level views (like SQL consoles or RLS editors), or replace any part of the UI with your own components. No black boxes.
