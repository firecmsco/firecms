In questa sezione dimostreremo alcuni casi d'uso avanzati che possono essere risolti con le molteplici funzionalità fornite da FireCMS.

Per esempio, possiamo costruire la struttura dati per un **blog** che puoi poi renderizzare usando SSR (server side rendering) con uno strumento come next.js per prestazioni velocissime e SEO ottimizzata.

In una delle ricette mostriamo come aggiungere logica personalizzata per copiare entità da una collezione all'altra.

Un altro problema comune è la gestione delle **localizzazioni**, quindi mostriamo come creare una struttura dati che funziona sia per gli sviluppatori che per i creatori di contenuti.
