## FireCMS PRO

Wenn Sie von früheren Beta-Versionen von FireCMS PRO migrieren, müssen Sie einige Aktualisierungen an Ihrem Projekt vornehmen.

### User Management Auth Controller

Für selbst gehostete Versionen hat sich die API für die Datenverwaltungs-Controller geändert. Der
`authController` wird jetzt an den User Management Controller übergeben, anstatt umgekehrt.

❌ Code vorher:
```typescript
    const userManagement = useBuildUserManagement({
        dataSourceDelegate: firestoreDelegate
    });

    const authController: FirebaseAuthController = useFirebaseAuthController({
        firebaseApp,
        signInOptions,
        loading: userManagement.loading,
        defineRolesFor: userManagement.defineRolesFor
    });
```

✅ Code nachher:

```typescript
    const authController: FirebaseAuthController = useFirebaseAuthController({
        firebaseApp,
        signInOptions
    });

    const userManagement = useBuildUserManagement({
        dataSourceDelegate: firestoreDelegate,
        authController
    });
```

Weitere Informationen zu den Hauptkomponenten finden Sie im Abschnitt [Hauptkomponenten](/docs/self/main_components).
