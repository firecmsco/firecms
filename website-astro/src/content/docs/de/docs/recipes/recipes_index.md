In diesem Abschnitt demonstrieren wir einige erweiterte Anwendungsfälle, die mit den
verschiedenen Funktionen von FireCMS gelöst werden können.

Zum Beispiel können wir die Datenstruktur für einen **Blog** aufbauen, den Sie
dann mit SSR (serverseitiges Rendering) mit einem Tool wie
next.js für blitzschnelle Performance und SEO rendern können.

In einem der Rezepte zeigen wir Ihnen, wie Sie benutzerdefinierte Logik hinzufügen, um Entities
von einer Kollektion in eine andere zu kopieren.

Ein weiteres häufiges Problem ist die Handhabung von **Lokalisierungen**, daher zeigen wir Ihnen,
wie Sie eine Datenstruktur erstellen, die sowohl für Entwickler als auch für Inhaltsersteller
funktioniert.
