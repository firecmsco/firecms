---

:::important
Sie müssen ein Dienstkonto nur dann manuell erstellen, wenn Sie FireCMS **keine** Administratorberechtigungen erteilen möchten. In der App können Sie ein neues Projekt erstellen oder bestehende Projekte verknüpfen, ohne manuell ein Dienstkonto erstellen zu müssen.
:::

Eine Möglichkeit, ein Projekt in FireCMS Cloud zu erstellen und es mit Ihrem **bestehenden** Firebase/GCP-Projekt zu verknüpfen, besteht darin, ein Dienstkonto zu erstellen, ihm die erforderlichen Berechtigungen zu erteilen und es Ihrem Projekt zuzuweisen.

Führen Sie dazu folgende Schritte aus:

- Gehen Sie zur [Google Cloud Console](https://console.cloud.google.com/).

- Wählen Sie das Projekt aus, das Sie mit FireCMS verknüpfen möchten.

- Gehen Sie zum Abschnitt [Dienstkonten](https://console.cloud.google.com/iam-admin/serviceaccounts).

<img
    loading="lazy"
    className={"ml-8 w-full rounded-lg mt-4 mb-8 max-w-[600px]"}
    src="/img/sa_creation/0_sa_menu.png"
    alt={"Dienstkonten-Menü"}/>

- Klicken Sie auf die Schaltfläche `Dienstkonto erstellen` (Create Service Account).

<img
    loading="lazy"
    className={"ml-8 w-full rounded-lg mt-4 mb-8 max-w-[800px]"}
    src="/img/sa_creation/1_create_new.png"
    alt={"Neues Dienstkonto erstellen"}/>

- Füllen Sie die Details des Dienstkontos aus. Nennen Sie es `FireCMS`.

<img
    loading="lazy"
    className={"ml-8 w-full rounded-lg mt-4 mb-8 max-w-[600px]"}
    src="/img/sa_creation/3_name.png"
    alt={"Details des Dienstkontos"}/>

- Weisen Sie folgende Rollen zu:
 - `Firebase Admin` (Firebase-Administrator)
 - `Firebase Admin SDK Administrator Service Agent`
 - `Firebase Service Management Service Agent`

<img
    loading="lazy"
    className={"ml-8 w-full rounded-lg mt-4 mb-8 max-w-[600px]"}
    src="/img/sa_creation/4_roles.png"
    alt={"Rollen des Dienstkontos"}/>

- Definieren Sie optional die Benutzer, die das Dienstkonto imitieren können.

<img
    loading="lazy"
    className={"ml-8 w-full rounded-lg mt-4 mb-8 max-w-[600px]"}
    src="/img/sa_creation/5_users.png"
    alt={"Benutzer des Dienstkontos"}/>

- Jetzt erstellen wir den JSON-Schlüssel, der zu FireCMS Cloud hochgeladen wird.
Finden Sie das neu erstellte Dienstkonto und klicken Sie im Dropdown-Menü auf `Schlüssel verwalten` (Manage keys).

<img
    loading="lazy"
    className={"ml-8 w-full rounded-lg mt-4 mb-8 max-w-[800px]"}
    src="/img/sa_creation/6_keys_menu.png"
    alt={"Schlüssel des Dienstkontos"}/>

- Erstellen Sie dann einen neuen Schlüssel.

<img
    loading="lazy"
    className={"ml-8 w-full rounded-lg mt-4 mb-8 max-w-[600px]"}
    src="/img/sa_creation/7_key_create.png"
    alt={"Schlüssel erstellen"}/>

- Und laden Sie schließlich den JSON-Schlüssel herunter.

<img
    loading="lazy"
    className={"ml-8 w-full rounded-lg mt-4 mb-8 max-w-[600px]"}
    src="/img/sa_creation/8_json.png"
    alt={"JSON-Schlüssel herunterladen"}/>

Jetzt können Sie diesen **JSON-Schlüssel** zu **FireCMS Cloud** hochladen und ihn mit Ihrem Projekt verknüpfen.

:::important[Sicherheit]
Ein Dienstkonto ist eine spezielle Art von Google-Konto, das es nicht-menschlichen Benutzern ermöglicht, Google Cloud Platform (GCP)-Dienste zu authentifizieren und zu autorisieren. Es ist wichtig, den Dienstkontoschlüssel sicher aufzubewahren, da er für den Zugriff auf Ihre GCP-Ressourcen verwendet werden kann. FireCMS Cloud verwendet dieses Dienstkonto zur Verwaltung der Ressourcen Ihres Projekts.
Ihr Dienstkonto wird sicher mit Google Cloud KMS verschlüsselt.
Stellen Sie sicher, dass Sie den Schlüssel Ihres Dienstkontos sicher aufbewahren und ihn nicht mit nicht autorisierten Benutzern teilen.
:::
