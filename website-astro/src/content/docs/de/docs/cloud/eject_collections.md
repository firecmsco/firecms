description: "Konvertieren Sie in der UI definierte Kollektionen in Code für Ihr Headless-CMS. Exportieren Sie Ihre Firestore-Schemasonfiguration für Versionskontrolle und erweiterte Anpassung."
---

Die **neueste Version von [FireCMS](https://app.firecms.co)**, Version 3.0, ermöglicht es Ihnen, das CMS vollständig in der Cloud zu verwalten, ohne Code verwalten zu müssen. Sie können Kollektionen mit unserer **UI zur Erstellung beliebiger Schemata** erstellen.

Betrachten Sie die Bücher-Kollektion aus unserer [Demo](https://demo.firecms.co). Die Kollektion enthält Felder wie Titel, Untertitel, Autoren, Bewertung, Kategorie usw. Sie können diese Kollektion über die UI erstellen und dann von der UI aus verwalten.

![edit_collection_in_ui](/img/docs/collections/eject_collections_book_collection.png)

Aber was, wenn Sie die Ansicht **weiter anpassen** möchten oder es vorziehen, Code zur Erstellung der Kollektion zu verwenden? FireCMS ermöglicht genau das. Sie können einen Anpassungsprozess mit unserem [Schnellstart-Leitfaden](https://firecms.co/docs/cloud/quickstart) starten und ihn in die Cloud hochladen, wobei Sie dieselbe gehostete App verwenden.

Eine häufige Frage, die wir erhalten, ist, wie man den Code einer Kollektion importiert. Es ist so einfach wie das Navigieren zum Schema-Editor, Klicken auf die Code-Schaltfläche und Kopieren des generierten Codes. Sie können dann **die Kollektion exportieren** und sie direkt aus Ihrer Codebasis heraus verwalten.

![edit_collection_code_button](/img/docs/collections/eject_collections_code_button.png)

Fügen Sie dann den kopierten Code in Ihren bevorzugten Code-Editor ein.

![edit_collection_code_button](/img/docs/collections/eject_collections_code.png)
