Nesta seção vamos demonstrar alguns casos de uso avançados que podem ser resolvidos
com os múltiplos recursos fornecidos pelo FireCMS.

Por exemplo, podemos construir a estrutura de dados para um **blog** que você pode
então renderizar usando SSR (server side rendering) com uma ferramenta como
next.js para desempenho extremamente rápido e SEO.

Em uma das receitas, mostramos como adicionar lógica personalizada para copiar entidades 
de uma coleção para outra.

Outro problema comum é lidar com **localizações**, então mostramos como
criar uma estrutura de dados que funcione tanto para desenvolvedores quanto para criadores de conteúdo.
