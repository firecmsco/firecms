Dans cette section, nous allons démontrer quelques cas d'utilisation avancés qui peuvent être résolus avec les multiples fonctionnalités fournies par FireCMS.

Par exemple, nous pouvons construire la structure de données pour un **blog** que vous pouvez ensuite rendre en utilisant SSR (rendu côté serveur) avec un outil comme next.js pour des performances ultra-rapides et le SEO.

Dans l'une des recettes, nous vous montrons comment ajouter une logique personnalisée pour copier des entités d'une collection à une autre.

Un autre problème courant est la gestion des **localisations**, donc nous vous montrons comment créer une structure de données qui fonctionne à la fois pour les développeurs et les créateurs de contenu.
