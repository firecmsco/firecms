Veuillez [nous contacter par email](mailto:hello@firecms.co),
ou [planifier un appel](https://calendar.google.com/calendar/u/0/appointments/schedules/AcZssZ0INW8ihjQ90S4gkdo8_rbL_Zx7gagZShLIpHyW43zDXkQDPole6a1coo1sT2O6Gl05X8lxFDlp?gv=true).
:::

Vous pouvez utiliser les fonctionnalités de FireCMS PRO localement pendant le développement sans licence, mais vous devrez en acheter une pour le déployer. Il y a une période de grâce après le premier déploiement pour vous permettre de le tester en production.

Vous pouvez acheter une licence sur le [site des abonnements FireCMS](https://app.firecms.co/subscriptions).

Lors de la création de votre licence, vous devez spécifier les IDs de projets qui utiliseront la licence. Vous pouvez trouver votre ID de projet dans la console Firebase.

Vous recevrez une clé API que vous devrez passer à votre composant `FireCMS`. Si vous utilisez un template de démarrage, vous pouvez la définir dans le fichier `.env`.
